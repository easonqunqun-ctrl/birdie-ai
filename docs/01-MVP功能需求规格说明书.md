## 二、全局约定

### 2.1 术语定义

| 术语 | 定义 |
|------|------|
| 挥杆分析 | 用户上传挥杆视频后，AI 对视频进行姿态估计、阶段分割、动作评分、问题诊断并生成报告的完整流程 |
| 分析报告 | 一次挥杆分析的完整输出，包含评分、雷达图、问题诊断、训练建议 |
| 分析次数 | 用户触发一次完整挥杆分析消耗的配额，不论分析成功或失败（失败退回） |
| 对话轮次 | 用户发送一条消息 + AI 回复一条消息 = 1 轮对话 |
| 会员 | 付费订阅用户，享有无限分析和对话权限 |
| 免费用户 | 未订阅的注册用户，受分析次数和对话轮次限制 |
| 训练任务 | AI 根据分析结果生成的单个练习动作项 |
| 打卡 | 用户标记完成某个训练任务 |

### 2.2 免费用户限制

| 功能 | 免费用户限制 | 会员 |
|------|-------------|------|
| 挥杆分析 | 每自然月 3 次 | 无限 |
| AI 对话 | 每日 5 轮对话 | 无限 |
| 训练计划 | 仅查看，不可自定义 | 完整功能 |
| 历史报告 | 保留最近 3 份 | 全部保留 |
| 进步曲线 | 不可用 | 可用 |
| 分析速度 | 标准队列 | 优先队列 |

### 2.3 通用交互规范

- **加载状态**：所有异步操作显示骨架屏或加载动画，超过 3 秒显示进度提示
- **错误提示**：统一使用 Toast 组件，显示 2 秒自动消失；严重错误使用 Modal 弹窗
- **空状态**：所有列表页面需设计空状态插画与引导文案
- **网络异常**：断网时显示"网络连接异常，请检查网络"+ 重试按钮
- **下拉刷新**：列表页面支持下拉刷新
- **触觉反馈**：关键操作（拍摄、提交、完成）触发轻微震动反馈

### 2.4 视觉与品牌规范

MVP 阶段所有 **用户可见界面**（小程序、App、分享卡片等）须遵循 [`小鸟AI高尔夫-产品设计白皮书.md`](../小鸟AI高尔夫-产品设计白皮书.md) **§7.2 产品视觉规范（权威，全端统一）**：

- **主色体系**：**靛蓝 + 白 + 金 + 点缀绿**（以 [`client/src/app.scss`](../client/src/app.scss) CSS 变量为权威源）：靛蓝 `#1a237e` 用于定调与主操作，白用于信息与卡片承载，金 `#c9a227` 用于小面积高价值强调（如会员、出色等级、关键数据），点缀绿 `#00d084` 仅用于「成长 / 完成 / 上行」语义；不得用金色铺满主界面，亦不得以高饱和浅绿 / 荧光绿 / Tailwind emerald 反客为主。

## 三、M1 用户体系

### 3.1 微信授权登录

**功能描述**：用户通过微信授权一键登录小程序，无需手动注册。

**前置条件**：用户已打开小程序。

**详细流程**：

```
1. 用户首次进入小程序
2. 展示欢迎页（品牌 Logo + 产品 Slogan + "微信用户一键登录"按钮）
3. 用户点击登录按钮
4. 调用 wx.login() 获取临时 code
5. 将 code 发送至后端，后端调用微信 code2session 接口获取 openid + session_key
6. 后端查找是否已有该 openid 的用户记录
   - 已有 → 返回 JWT Token + 用户信息，跳转首页
   - 没有 → 创建新用户记录，返回 JWT Token，进入新用户引导流程
7. 前端存储 Token 至本地缓存（wx.setStorageSync）
```

**字段要求**：

| 字段 | 来源 | 是否必填 | 说明 |
|------|------|----------|------|
| openid | 微信 API | 自动 | 用户唯一标识 |
| union_id | 微信 API | 自动（如有） | 跨平台标识 |
| nickname | 用户输入 | 必填（引导填写） | 2-12 个字符 |
| avatar | 微信头像 / 用户上传 | 选填 | 默认使用微信头像 |

**异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| wx.login 失败 | Toast 提示"登录失败，请重试"，提供重试按钮 |
| 后端 code2session 失败 | Toast 提示"服务暂时不可用"，3 秒后自动重试一次 |
| Token 过期 | 静默刷新 Token，刷新失败则弹出重新登录提示 |
| 用户拒绝微信授权 | 显示引导页说明需要授权的原因，提供"重新授权"按钮 |

**验收标准**：

- [ ] 新用户首次登录全流程 < 3 秒（不含网络延迟） <!-- 工程排期：[docs/19 §6.5](./19-产品开发迭代计划-当前队列.md#65-docs01-未勾--部分完成--plan-id对照表) **O-01**；真机性能验收放在 W2 收官前联调 -->
- [x] 已有用户登录（Token 有效）自动进入首页，无感知（`login` 页 `useDidShow` 守卫 + `bootstrap` 恢复）
- [x] Token 有效期 7 天，支持静默续期（`/v1/auth/refresh-token` + `request.ts` 近过期自动换签）
- [x] 登录态丢失后可一键重新登录（`userStore.logout` 清 token，`login` 页常驻可达）

### 3.2 新用户引导（Onboarding）

**功能描述**：新用户首次登录后，通过 3 个简短问题建立高尔夫档案。

**引导问题设计**：

```
问题 1/3：你打球多久了？
  ○ 刚开始学（0-6 个月）
  ○ 有一些基础（6 个月-2 年）
  ○ 经常打（2-5 年）
  ○ 老球骨了（5 年以上）

问题 2/3：你最想改善什么？
  ☐ 击球距离
  ☐ 方向准确性
  ☐ 短杆技术
  ☐ 推杆
  ☐ 整体稳定性
  （可多选，至少选 1 项）

问题 3/3：你每周大概练几次球？
  ○ 偶尔练（每月 1-2 次）
  ○ 每周 1 次
  ○ 每周 2-3 次
  ○ 每周 4 次以上
```

**交互规范**：

- 每个问题独占一屏，左滑/点击"下一步"切换
- 支持"返回"修改上一题答案
- 第 3 题完成后显示"完成"按钮，点击后进入首页
- 底部显示"跳过"链接（跳过后档案字段为空，后续在"我的"页面可补填）
- 页面底部显示进度指示器（1/3, 2/3, 3/3）

**数据映射**：

| 问题 | 存储字段 | 值映射 |
|------|----------|--------|
| 打球多久 | golf_level | beginner / elementary / intermediate / advanced |
| 想改善什么 | primary_goals | 数组，如 ["distance", "accuracy"] |
| 每周练几次 | weekly_practice_frequency | occasional / once / frequent / daily |

**验收标准**：

- [x] 3 个问题流程可在 30 秒内完成（单屏一问 + 进度条，后端实测 P95 < 150ms）
- [x] 跳过引导后用户可正常使用所有功能（`PATCH /v1/users/me` 仅置 `onboarding_completed=true`）
- [x] 引导数据正确存储至用户档案（`POST /v1/users/me/onboarding` 写入 `golf_level / primary_goals / weekly_practice_frequency`）
- [x] 已完成引导的用户再次登录不再显示引导（`login` 页 `useDidShow` + `onboarding_completed` 联合判断）

### 3.4 账号注销

**功能描述**：用户可在"设置"页面申请注销账号，满足隐私合规要求（PIPL）。

**入口**：我的 → 设置 → 注销账号

**详细流程**：

```
1. 用户点击"注销账号"
2. 展示注销须知页面：
   - 账号注销后所有数据将被永久删除
   - 注销申请提交后有 7 天冷静期
   - 冷静期内可随时登录取消注销
   - 如有未完成的会员/订单将无法注销
3. 用户确认 → 输入 "DELETE" 二次确认
4. 提交注销申请 → 进入 7 天冷静期
5. 页面显示"账号将于 {date} 注销，登录可取消"
6. 冷静期到期后：
   - 软删除用户记录
   - 删除对象存储中的视频文件
   - 匿名化分析数据（保留统计价值，去除个人标识）
   - 删除对话记录
   - 清除支付流水中的敏感信息
```

**异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| 存在未完成订单 | 拒绝注销，提示"你有未完成的订单，请处理后再注销" |
| 冷静期内登录 | 弹出"你的账号正在注销中，是否取消？"+ 取消注销按钮 |
| 注销后扫码进入 | 视为全新用户，重新创建账号 |

**验收标准**：

- [x] 注销流程包含二次确认（输入 `DELETE` + `showModal` 最后确认；`client/src/pages/profile/account-deletion.tsx` + `POST` 带确认文案）
- [x] 7 天冷静期内可成功取消注销（`cancelAccountDeletion` + `account_deletion_service.cancel_account_deletion`）
- [x] 冷静期到期后数据正确清理 —— **双路径**：
  - **懒清理**：`get_user_by_id` 在用户下次发请求时触发（`backend/app/services/user_service.py`）
  - **Beat 兜底**：`xiaoniao.purge_due_account_deletions` 每小时扫一次（`backend/app/tasks/account_tasks.py` + `celery_app.beat_schedule`），用户即便不再登录也按 PIPL 承诺如期清掉；任务挂 `time_limit=30min` / `soft_time_limit=25min` 防历史积压被默认 5 分钟 kill
  - 回归保护：`backend/tests/test_account_deletion.py` 覆盖申请 happy/错误确认/冷静期幂等 / 取消 happy/未排期拒绝 / 懒清理到期/未到期 / beat 清理 happy/异常计数/noop / beat 注册 / task 注册 等 **12 个用例**（`backend-pytest-smoke` CI 真 PG+Redis 跑）
- [x] 注销后重新进入视为新用户（新 `openid` 创建新用户；**真机联调**仍建议季度抽测）

> **par-C1 验收纪要**：详见 [`docs/release-notes/par-C1-account-deletion-acceptance-memo.md`](./release-notes/par-C1-account-deletion-acceptance-memo.md)（4 条 §3.4 验收 + 5 条隐含项的逐条证据链 + 测试覆盖矩阵 + 上线观测告警建议 + 三方复核栏）。

### 3.5 会员过期与配额降级

**功能描述**：会员到期后自动降级为免费用户，配额规则平滑过渡。

**降级规则**：

| 维度 | 规则 | 说明 |
|------|------|------|
| 分析配额 | 当月已有记录保持不变，下个自然月恢复为 3 次/月 | 避免到期日当天突然限制 |
| 对话配额 | 次日起恢复为 5 轮/天 | 以自然日为界 |
| 历史报告 | 全部保留可查看，但不再新增（超过 3 份后锁定"上传新分析"） | 已有数据不删除 |
| 进步曲线 | 锁定，显示会员续费引导 | 保留数据，续费后恢复 |
| 训练计划 | 已有计划可继续查看/打卡，不再生成新的个性化计划 | 渐进过渡 |
| 分析优先队列 | 次日起恢复为标准队列 | — |

**到期提醒策略**：

```
会员到期前 7 天：小程序内消息提醒（非侵入式）
会员到期前 3 天：微信服务通知 + 小程序内弹窗
会员到期前 1 天：微信服务通知（含续费链接）
会员到期当天：登录时弹出挽留页面（显示使用数据摘要 + 优惠续费价格）
```

**验收标准**：

- [x] 会员到期后权益正确降级（`payment_service.ensure_membership_valid` 惰性检查：`membership_tier` 置 `free` 且配额从无限回置为免费档）
- [x] 当月分析配额不受影响（回置为 `3` 次/月等免费规则，不按「清空当月已用」处理；见 `payment_service` / `quota_service`）
- [x] 历史数据全部保留（不删 `swing_analyses` 等业务行；仅权限与新增能力受限，见前文表格）
- [~] 到期提醒按时发送 —— **已过期**：§1.4.2；**到期前多档**：Celery `membership_pre_expiry_notify`（csv `7,3,1` 可配）+ Redis 按档去重 + 会员页订阅授权 + **站内弹窗兜底**（[`useMembershipExpiringSoonModal`](../client/src/hooks/useMembershipExpiringSoonModal.ts)）；详见 [`docs/02` §1.4.3](./02-API接口设计文档.md#143-微信小程序--会员到期前-n-天提醒celery-beat非-rest) <!-- **DOC-02** · Batch-B ✅ -->
- [x] 续费后权益立即恢复（支付成功链路 `activate_membership` → 配额 `total=-1` 等）

### 3.6 示例视频体验

**功能描述**：新用户（或无分析记录的用户）可使用预置示例视频体验完整分析流程，降低首次使用门槛。

**入口**：拍摄引导页底部"用示例视频体验一下"

**详细规则**：

| 规则 | 说明 |
|------|------|
| 触发条件 | 用户分析记录为 0 时显示入口 |
| 示例视频数量 | MVP 预置 2 个（正面角度 + 侧面角度各 1 个） |
| 配额消耗 | 不消耗免费分析次数 |
| 分析报告 | 生成完整报告，标记为"示例体验" |
| 报告保留 | 不计入历史报告列表，不影响进步曲线 |
| 可重复体验 | 否，每个示例视频仅可体验 1 次 |

**示例视频管理**：
- 示例视频由运营后台配置，存储在 COS 特定路径

| 异常 | 处理方式 |

| 分析超时（> 120 秒未完成） | 提示"分析时间较长，我们会在完成后通知你"，允许离开 |

### 4.4 视频质量检测

**功能描述**：在正式分析前对视频进行质量预检，降低分析失败率。

**检测项**：

| 检测项 | 判定标准 | 结果处理 |
|--------|----------|----------|
| 人体检测 | 至少检测到 1 个完整人体（头-脚可见） | 未检测到 → 阻断，提示重新拍摄 |

| 遮挡检测 | 人体关键部位无严重遮挡 | 部分遮挡 → 警告；严重遮挡 → 阻断 |
