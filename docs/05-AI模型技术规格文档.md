### 2.1 分析流水线架构

```
原始视频 → 预处理 → 姿态估计 → 挥杆检测 → 阶段分割
                                                │

```

**整体延迟目标**：端到端 < 30 秒（标准队列）/ < 15 秒（优先队列）

### 2.2 模块 ①：视频预处理

**输入**：用户上传的原始视频文件（MP4/MOV）

**处理步骤**：

| 步骤 | 操作 | 参数 | 说明 |
|------|------|------|------|
| 1 | 格式转码 | → MP4 (H.264) | 统一格式 |
| 2 | 帧率标准化 | → 30 fps | 统一时序基准 |
| 3 | 分辨率调整 | → 短边 720px，保持比例 | 平衡精度和速度 |
| 4 | 去抖动 | OpenCV stabilization | 减少相机抖动干扰 |
| 5 | 亮度/对比度归一化 | 自适应直方图均衡 | 适应不同光照 |
| 6 | 帧提取 | 全帧提取 + 关键帧标记 | 供后续模块使用 |

**输出**：

```python
class PreprocessResult:
    frames: list[np.ndarray]       # 所有帧 (H, W, 3)

    total_frames: int

```

**工具选型**：OpenCV 4.x + FFmpeg

**性能目标**：10 秒视频预处理 < 3 秒

**模型选型**：

| 方案 | 精度 | 速度 | 选择 |

**输出**：

```python
class PoseResult:
```

**质量阈值**：

**性能目标**：10 秒视频（300 帧）姿态估计 < 5 秒（GPU）/ < 10 秒（CPU）

### 2.4 模块 ③：挥杆检测与阶段分割

**挥杆检测**：

从连续帧中定位挥杆动作的起止点，丢弃非挥杆帧。

**检测方法**：

```
1. 计算手腕关键点（15, 16）的帧间位移速度
2. 速度 > 阈值的连续帧区间 = 候选挥杆区间
3. 选择最长 / 速度最大的候选区间为目标挥杆
4. 向前后各扩展 0.5 秒（包含准备和收杆）
```

**阶段分割**（6 阶段）：

| 阶段 | 英文 | 检测特征 |
|------|------|----------|
| 准备 | Setup / Address | 身体静止，双手在球前方 |
| 上杆 | Backswing | 手腕位置上升，肩部开始旋转 |
| 顶点 | Top | 手腕达到最高点，肩旋转最大 |
| 下杆 | Downswing | 手腕快速下降，髋部开始旋转 |
| 击球 | Impact | 手腕到达最低点附近 |
| 收杆 | Follow-through | 击球后手腕继续上升至收杆位 |

**分割算法**：

```
MVP 方案：基于规则 + 关键点速度/位置启发式

1. Top 点检测：手腕 Y 坐标（左手腕或右手腕，取决于正/反手）
   到达最高位的帧 = Top 帧
2. Impact 点检测：下杆阶段手腕速度最大 + Y 坐标最低的帧
3. Setup / Follow-through：挥杆区间的首/尾
4. Backswing = Setup 至 Top
5. Downswing = Top 至 Impact

后期方案：训练 1D 时序分类模型（LSTM / Transformer）
```

**输出**：

```python
class PhaseSegmentResult:
    phases: dict[str, PhaseInfo]
    # PhaseInfo: start_frame, end_frame, start_time, end_time, key_frame_idx

    swing_detected: bool
    swing_start_frame: int
    swing_end_frame: int
    handedness: str              # "right" / "left" 自动检测左右手
```

**精度目标**：

- 挥杆检测成功率 > 95%（有效视频）
- 阶段分割帧级误差 < ±3 帧（100ms @30fps）

从骨骼关键点时序数据中提取高尔夫挥杆关键生物力学特征。

**特征清单（MVP 阶段 15 个核心特征）**：

| # | 特征名 | 计算方法 | 评分阶段 | 单位 |
|---|--------|----------|----------|------|
| 1 | 准备脊柱前倾角 | 肩中点-髋中点与垂直线的夹角 @Setup | Setup | 度 |
| 2 | 准备膝弯角 | 膝关节角度 @Setup | Setup | 度 |

| 6 | 左臂伸直度 | 左肘关节角度 @Top | Top | 度 |
| 7 | 顶点手腕位置 | 手腕相对于头部的位置 @Top | Top | 比例 |

| 14 | 收杆高度 | 收杆时手腕位置高度 | Follow-through | 比例 |
| 15 | 收杆平衡 | 收杆结束后身体稳定性（关键点抖动量） | Follow-through | 像素 |

**角度计算方法**：

```python
def calculate_angle(point_a, point_b, point_c) -> float:
    """计算三点形成的角度（point_b 为顶点）"""
    ba = point_a - point_b
    bc = point_c - point_b
    cosine = np.dot(ba, bc) / (np.linalg.norm(ba) * np.linalg.norm(bc) + 1e-8)
    return np.degrees(np.arccos(np.clip(cosine, -1.0, 1.0)))

def calculate_rotation_angle(left_point, right_point, reference_line="horizontal") -> float:
    """计算两点连线与参考线的夹角（用于肩/髋旋转）"""
    diff = right_point - left_point
    angle = np.degrees(np.arctan2(diff[1], diff[0]))
    return angle
```

**输出**：

```python
class FeatureResult:

    # {
    #   "value": 35.0,
    #   "frame_idx": 45,
    #   "confidence": 0.9,
    #   "unit": "degrees"
    # }
```

### 2.6 模块 ⑤：动作评分

**评分逻辑**：将用户特征值与职业球手参考范围对比，计算偏差得分。

**职业球手参考数据（初始值，持续校准）**：

| 特征 | 职业均值 | 理想范围 | 权重（在所属阶段内） |
|------|----------|----------|---------------------|
| 准备脊柱前倾角 | 30° | 25°-35° | 0.35 |
| 准备膝弯角 | 155° | 150°-165° | 0.30 |
| 肩旋转角（上杆） | 90° | 80°-100° | 0.35 |
| 髋旋转角（上杆） | 45° | 35°-55° | 0.25 |
| X-Factor | 45° | 35°-55° | 0.20 |
| 左臂伸直度 | 175° | 165°-180° | 0.50 |
| 顶点手腕位置 | 头顶上方 | — | 0.50 |
| 下杆顺序指标 | 髋先于肩 3-5 帧 | 2-7 帧 | 0.35 |
| 手腕释放角 | — | — | 0.30 |
| 手腕释放时机 | 下杆后 60% | 50%-70% | 0.35 |
| 击球脊柱前倾角 | 与准备位一致 | ±5° | 0.50 |
| 头部位移量 | < 5cm | 0-8cm | 0.50 |
| 挥杆节奏比 | 3:1 | 2.5:1 - 3.5:1 | 0.40 |
| 收杆高度 | 肩上方 | — | 0.50 |
| 收杆平衡 | < 阈值 | — | 0.50 |

**评分算法**：

```python
def score_feature(value: float, ideal_min: float, ideal_max: float,
                  tolerance: float = 0.5) -> int:
    """
    单特征评分（0-100）
    - 在理想范围内 → 85-100
    - 偏离理想范围但在容忍范围内 → 60-84
    - 偏离容忍范围 → 0-59
    """
    if ideal_min <= value <= ideal_max:
        center = (ideal_min + ideal_max) / 2
        range_half = (ideal_max - ideal_min) / 2
        deviation = abs(value - center) / range_half  # 0-1
        return int(100 - deviation * 15)  # 85-100

    if value < ideal_min:
        deviation = (ideal_min - value) / (ideal_max - ideal_min)
    else:
        deviation = (value - ideal_max) / (ideal_max - ideal_min)

    deviation = min(deviation / tolerance, 1.0)
    return int(84 - deviation * 84)  # 0-84

def score_phase(features: dict, phase_config: list) -> int:
    """阶段评分 = 加权特征评分之和"""
    total = sum(
        score_feature(features[f["name"]], f["ideal_min"], f["ideal_max"]) * f["weight"]
        for f in phase_config
        if f["name"] in features
    )
    return round(total)

def score_overall(phase_scores: dict) -> int:
    """综合评分 = 加权阶段评分之和"""
    weights = {
        "setup": 0.15, "backswing": 0.20, "top": 0.15,
        "downswing": 0.25, "impact": 0.15, "follow_through": 0.10
    }
    return round(sum(phase_scores[p] * w for p, w in weights.items()))
```

### 2.7 模块 ⑥：问题诊断

**MVP 覆盖 5 种常见错误模式**：

| # | 错误类型 | 英文 | 检测依据 | 严重度判定 |
|---|----------|------|----------|-----------|
| 1 | 抛杆 | Casting | 手腕释放时机 < 40%（过早） | 偏离越大越严重 |
| 2 | 由外到内下杆 | Over-the-top | 下杆初期手部轨迹方向（需 down_the_line 视角） | 偏离角度 |
| 3 | 提前伸展 | Early Extension | 击球时髋部前移量（脊柱角变化） | 变化量 |
| 4 | 侧移 | Sway / Slide | 上杆时髋部水平位移 > 阈值 | 位移量 |
| 5 | 失去身体姿势 | Loss of Posture | 挥杆过程中脊柱角度变化 > 阈值 | 变化量 |

**诊断算法（MVP：基于规则引擎）**：

```python
class IssueDiagnoser:
    def diagnose(self, features: FeatureResult, phases: PhaseSegmentResult) -> list[Issue]:
        issues = []

        # 1. Casting 检测
        release_timing = features["wrist_release_timing"]
        if release_timing < 0.40:
            severity = "high" if release_timing < 0.25 else "medium"
            issues.append(Issue(
                type="casting",
                name="抛杆（Casting）",
                severity=severity,
                confidence=min(1.0, (0.40 - release_timing) / 0.20),
                description=self._gen_casting_description(release_timing),
                metrics={"wrist_release_timing": release_timing, "ideal_range": [0.50, 0.70]}
            ))

        # 2. Early Extension 检测
        spine_change = abs(features["spine_angle_impact"] - features["spine_angle_setup"])
        if spine_change > 8:
            severity = "high" if spine_change > 15 else "medium" if spine_change > 10 else "low"
            issues.append(Issue(
                type="early_extension",
                ...
            ))

        # ... 类似逻辑处理其他错误类型

        # 按严重度排序
        severity_order = {"high": 0, "medium": 1, "low": 2}
        issues.sort(key=lambda x: severity_order[x.severity])

        return issues
```

**置信度**：每个诊断结果附带 confidence 值（0-1），< 0.6 的诊断不展示给用户（MVP 阶段保守策略）。

**后期升级路径**：规则引擎 → 训练 ML 分类器（使用标注数据） → 端到端深度学习模型

## 五、降级与容错策略

| 场景 | 降级方案 |
|------|----------|
| 姿态估计失败 | 返回"视频分析失败"，退回配额 |
| 阶段分割失败 | 使用全视频分析（不分阶段），标注"仅供参考" |

| GPU 资源不足 | 将任务放入等待队列，增加预估等待时间 |

### 8.2 阶段分割算法 V2（对应 §2.4）

| 当前痛点 | V2 算法 |
|---------|--------|
| 单一启发式（`MIN_MOTION_SPEED=0.008`）不适配慢挥 / 大幅试挥 | 多信号融合：主手腕速度峰 + 杆头位置（若杆识别）+ 双脚 y 压力变化代理 + 髋部旋转速度 |
| 试挥被识别为正式挥 | 完整性检测：必须 setup→top→impact→follow 完整序列 |
| 短视频经常 `NoSwingError` | `MIN_DURATION_SEC` 从 2.0 → 1.5 |
| 长视频只识别第一次 | 多挥杆分割：识别所有挥杆区间，让用户选 |

实现路径：

1. 保留 V1 启发式作为 fallback
2. 训练轻量 1D CNN（或 Transformer-tiny）做软概率
3. ECS v2 提供阶段边界标注作为训练 / 验证集
4. 推理：神经网络给软概率 → 启发式做硬约束（top 必须在 setup 与 impact 之间）

### 8.3 新特征矩阵（对应 §2.5 升级）

| 新特征 | 计算方法 | 期望区间 / 标准 |
|--------|---------|---------------|
| `tempo_ratio` | backswing_frames / downswing_frames | 期望 3:1（pro 标准），容忍 2.5-3.5 |
| `tempo_consistency` | 多次挥杆的 tempo 标准差 | 单次回退为"是否在合理区间" |
| `pressure_shift_quality` | 用足部 y / 髋部 x 偏移代理 | backswing 重心右、downswing 左、follow 左 |
| `kinematic_sequence_quality` | 4 段速度峰时差（pelvis→torso→arms→wrists） | 时差递增；每段 60-100ms |
| `head_stability` | 鼻关键点在整段挥杆中的位移方差 | 位移 < 20px @ 720p |

新特征写入 `swing_analyses.new_features_payload`（详 docs/03 §9.1）；进入报告"动力学详情"折叠区，避免一次性吓退用户。

### 8.4 球杆差异化标尺（对应 §2.5 / §2.6 升级）
