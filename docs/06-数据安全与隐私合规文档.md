## 二、数据分类分级

### 2.1 数据分类

| 数据类别 | 包含数据 | 安全等级 | 存储要求 |
|----------|----------|----------|----------|
| **用户身份数据** | OpenID、昵称、头像、手机号（如有） | 高 | 加密存储，访问审计 |
| **用户行为数据** | 登录记录、页面访问、功能使用 | 中 | 脱敏存储 |
| **挥杆视频数据** | 用户上传的挥杆视频文件 | 高 | 加密存储，访问控制 |
| **AI 分析数据** | 骨骼数据、评分、诊断结果 | 中 | 常规存储 |
| **对话数据** | 用户与 AI 教练的对话记录 | 高 | 加密存储 |
| **支付数据** | 订单信息（不含银行卡/密码） | 高 | 加密存储，合规审计 |
| **设备数据** | 设备型号、系统版本、网络类型 | 低 | 常规存储 |

### 2.2 敏感数据识别

| 数据字段 | 敏感等级 | 处理方式 |
|----------|----------|----------|
| wechat_openid | 高 | AES-256 加密存储 |
| wechat_unionid | 高 | AES-256 加密存储 |
| 用户头像 | 中 | COS 私有读 + 签名 URL |
| 挥杆视频 | 高 | COS 私有读 + 服务端签名 + 自动过期 |
| 对话内容 | 高 | AES-256 加密存储 |
| 支付订单号 | 高 | 日志中脱敏显示 |
| IP 地址 | 中 | 仅存储用于安全风控，30 天后删除 |

## 三、数据收集与使用规范

### 3.1 收集的数据清单

| 数据项 | 收集目的 | 收集方式 | 是否必要 | 用户可否拒绝 |
|--------|----------|----------|----------|-------------|
| 微信 OpenID | 账号体系 | 微信授权 | 必要 | 否（拒绝则无法使用） |
| 昵称 | 个性化展示 | 用户主动填写 | 非必要 | 是 |
| 头像 | 个性化展示 | 微信授权或上传 | 非必要 | 是 |
| 高尔夫档案（水平/目标/频率） | AI 个性化推荐 | 用户主动填写 | 非必要 | 是（可跳过） |
| 挥杆视频 | 核心功能（AI 分析） | 用户主动上传 | 必要（使用分析功能时） | 是（可不用此功能） |
| 对话内容 | AI 对话功能 | 用户主动输入 | 必要（使用对话功能时） | 是 |
| 设备信息 | 兼容性和调试 | 自动采集 | 非必要 | 否 |
| 使用行为 | 产品优化 | 自动采集 | 非必要 | 是（可在设置中关闭） |

### 3.2 数据使用范围

| 使用目的 | 使用的数据 | 是否需额外授权 |
|----------|-----------|---------------|
| 提供 AI 挥杆分析服务 | 挥杆视频 | 否（核心功能授权覆盖） |
| 提供 AI 对话教练服务 | 对话内容 + 分析历史 | 否（核心功能授权覆盖） |
| 生成个性化训练计划 | 分析数据 + 用户档案 | 否 |
| 训练/优化 AI 模型 | 脱敏后的视频和分析数据 | **是（需用户额外同意）** |
| 产品数据统计（聚合） | 使用行为（匿名聚合） | 否 |
| 发送服务通知 | 微信 OpenID | 否（通过微信模板消息） |

### 3.3 禁止的数据使用行为

- 禁止将用户个人信息出售给第三方
- 禁止将用户挥杆视频用于非授权用途
- 禁止在未脱敏的情况下将用户数据用于模型训练
- 禁止向 LLM 传输可识别用户真实身份的信息
- 禁止在公开场合展示可识别的用户数据

## 四、数据存储安全

### 4.1 加密方案

| 层级 | 方案 | 说明 |
|------|------|------|
| 传输加密 | TLS 1.3 | 所有 API 通信强制 HTTPS |
| 数据库敏感字段加密 | AES-256-GCM | OpenID、对话内容等 |
| 密钥管理 | 腾讯云 KMS | 加密密钥托管 |
| 文件存储加密 | COS 服务端加密（SSE-KMS） | 视频和图片文件 |
| 数据库连接 | SSL/TLS | PostgreSQL 连接加密 |

### 4.2 应用层加密实现

```python
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

class FieldEncryptor:
    """敏感字段加密/解密工具"""

    def __init__(self, key: bytes):
        self.aesgcm = AESGCM(key)

    def encrypt(self, plaintext: str) -> str:
        nonce = os.urandom(12)
        ciphertext = self.aesgcm.encrypt(nonce, plaintext.encode(), None)
        return base64.b64encode(nonce + ciphertext).decode()

    def decrypt(self, encrypted: str) -> str:
        data = base64.b64decode(encrypted)
        nonce, ciphertext = data[:12], data[12:]
        return self.aesgcm.decrypt(nonce, ciphertext, None).decode()
```

### 4.3 访问控制

| 数据类型 | 访问权限 | 控制方式 |
|----------|----------|----------|
| 用户个人信息 | 仅本人 + 管理员 | API 层 user_id 校验 |
| 挥杆视频 | 仅本人 | COS 私有 + 临时签名 URL（有效期 1 小时） |
| 分析报告 | 仅本人（分享除外） | API 层 user_id 校验 |
| 对话记录 | 仅本人 | API 层 user_id 校验 |
| 支付信息 | 仅本人 + 财务管理员 | 角色权限控制 |
| 聚合统计 | 管理员 | 后台管理系统 RBAC |

## 五、数据生命周期管理

### 5.1 数据保留策略

| 数据类型 | 活跃用户保留期 | 非活跃用户保留期 | 注销用户 |
|----------|--------------|-----------------|----------|
| 用户基本信息 | 永久 | 账号非活跃 2 年后提醒 | 30 天内删除 |
| 挥杆视频（原始） | 会员：永久；免费：90 天 | 180 天后删除 | 30 天内删除 |
| 分析数据 | 永久 | 2 年 | 30 天内删除 |
| 骨骼数据 | 永久 | 2 年 | 30 天内删除 |
| 对话记录 | 180 天 | 90 天 | 30 天内删除 |
| 支付记录 | 永久（财务法规要求） | 永久 | 脱敏保留 |
| 设备/行为日志 | 90 天 | 90 天 | 90 天 |
| 埋点数据 | 90 天 | 90 天 | 90 天后自动删除 |

### 5.2 数据删除流程

**用户主动注销**：

```
1. 用户在"设置"中点击"注销账号"
2. 显示注销须知：
   - 注销后所有数据将在 30 天内永久删除
   - 注销后无法恢复
   - 剩余会员时间不予退款
3. 要求二次确认（输入"确认注销"）
4. 后端标记账号为"待注销"状态
5. 7 天冷静期（期间可登录撤销注销）
6. 冷静期后 23 天内完成数据删除：
   - 删除 COS 中所有视频/图片文件
   - 删除数据库中所有个人数据
   - 支付记录脱敏保留（法规要求）
   - 删除向量数据库中的对话索引
7. 记录删除审计日志
```

## 七、AI 安全合规

### 7.1 生成式 AI 合规要求

依据《生成式人工智能服务管理暂行办法》：

| 要求 | 我们的措施 |
|------|-----------|
| 内容安全 | AI 回复经过安全过滤，屏蔽违规内容 |
| 标识义务 | 所有 AI 生成内容标注"AI 生成" |
| 投诉机制 | 提供"反馈"入口，用户可举报 AI 不当回复 |
| 数据训练合规 | 训练数据不含违法信息，获得授权 |
| 算法备案 | 如触及算法推荐监管，按要求备案 |

### 7.3 AI 对话安全边界

| 场景 | 处理方式 |
|------|----------|
| 用户询问伤病/医疗 | AI 回复安全提示，建议就医，不做诊断 |
| 用户情绪激动/负面 | AI 以关怀态度回应，必要时引导至人工客服 |
| 用户发送违规内容 | 不回复，记录并触发风控告警 |

| 用户试图越狱 Prompt | System Prompt 加入防护指令，拒绝角色扮演请求 |

## 八、第三方数据共享

### 8.1 数据共享方

| 第三方 | 共享数据 | 共享目的 | 安全措施 |
|--------|----------|----------|----------|
| 微信（腾讯） | OpenID、支付信息 | 登录和支付 | HTTPS + 微信加密方案 |
| 腾讯云 COS | 视频文件 | 文件存储 | SSE 加密 + 私有访问 |

| 数据分析工具 | 行为数据（匿名） | 产品分析 | 聚合数据，不含个人标识 |

### 8.2 传输给 LLM 的数据规范

**必须遵守**：

- 不传输用户 OpenID、手机号等身份信息
- 用户昵称用"用户"替代
- 分析数据仅传数值和类型，不传视频 URL
- 所有传输走 HTTPS
- 使用 DeepSeek 等国内 API，数据不出境

**Prompt 中的数据脱敏示例**：

```python
def sanitize_for_llm(user_analysis: dict) -> dict:
    """传给 LLM 前脱敏处理"""
    return {
        "golf_level": user_analysis["golf_level"],
        "overall_score": user_analysis["overall_score"],
        "phase_scores": user_analysis["phase_scores"],
        "issues": [
            {"type": i["type"], "severity": i["severity"], "metrics": i["metrics"]}
            for i in user_analysis["detected_issues"]
        ],
        # 不传：user_id, openid, video_url, nickname, avatar
    }
```

## 九、安全事件响应

### 9.1 安全事件分级

| 级别 | 描述 | 示例 | 响应时间 |
|------|------|------|----------|
| P0 — 严重 | 大规模数据泄露 | 数据库被脱库、视频批量泄露 | 立即响应 |
| P1 — 高 | 小范围数据泄露或服务被攻击 | 单用户数据暴露、API 被注入 | 1 小时内 |
| P2 — 中 | 安全漏洞发现（未被利用） | 权限校验遗漏、敏感信息日志泄露 | 24 小时内 |
| P3 — 低 | 安全配置问题 | 未开启 HTTPS、弱密码 | 72 小时内 |

### 9.2 响应流程

```
发现安全事件
    │
    ▼
判断严重等级（P0/P1/P2/P3）
    │
    ├── P0/P1：立即通知负责人 → 紧急处置 → 通知受影响用户 → 上报监管
    │
    ├── P2：记录 → 排期修复 → 验证
    │
    └── P3：记录 → 常规修复
    │
    ▼
事后复盘 → 更新安全措施 → 归档
```

## 十、用户协议与隐私政策要点

### 10.1 用户协议核心条款

1. **服务内容**：AI 挥杆分析、AI 对话教练、训练计划等
2. **使用限制**：禁止上传非本人或他人隐私视频、禁止恶意攻击
3. **知识产权**：分析报告归用户所有，AI 模型归平台所有
4. **免责声明**：AI 分析结果仅供参考，不替代专业教练指导
5. **费用说明**：会员订阅和退款政策
6. **帐号注销**：用户有权随时注销
7. **争议解决**：协商 → 仲裁

### 10.2 隐私政策核心条款

1. **信息收集**：明确列出收集的每项数据及用途
2. **信息使用**：仅用于提供服务和改善产品
3. **信息存储**：中国境内存储，加密保护
4. **信息共享**：不出售个人信息，必要第三方共享说明
5. **信息安全**：采取的安全措施
6. **用户权利**：查询、更正、删除、注销的权利和方式
7. **儿童保护**：14 岁以下需监护人同意
8. **Cookie/本地存储**：使用说明
9. **政策更新**：更新通知方式
10. **联系方式**：数据保护负责人联系方式

### 10.3 未成年人保护补充条款

针对青少年用户（P1 用户群）：

| 措施 | 说明 |
|------|------|
| 年龄确认 | 注册时询问年龄段 |
| 监护人授权 | < 14 岁需提供监护人手机号并获得验证 |
| 使用时长提醒 | 连续使用超 40 分钟弹出提醒 |
| 付费限制 | 未成年用户付费需监护人确认 |
| 内容安全 | AI 对话增加未成年保护过滤 |

## 十二、W8-T1 微信隐私指引运行时落地（2026-04-22 更新）

### 12.1 背景

微信 2023.9 起要求所有涉及隐私 API（`wx.login` / `wx.chooseMedia` / `wx.getLocation`
等）的小程序必须：

1. 在微信公众平台后台填写并提交《小程序用户隐私保护指引》
2. 在客户端运行时挂接微信官方接口，确保用户在首次调用隐私 API 时看到授权弹窗

第 1 步由产品/运营在公众平台后台完成；第 2 步由 W8-T1 在代码侧实现。

### 12.2 代码落地

| 位置 | 作用 |
|---|---|
| `client/src/utils/privacy.ts` | 封装 `ensurePrivacyAuthorized(apiName)` + `registerPrivacyAuthorizationHandler()`；自动兼容非 weapp / 老基础库 |
| `client/src/app.tsx` | `componentDidMount` 调 `registerPrivacyAuthorizationHandler()` 挂全局监听 |
| `client/src/adapters/login.ts::loginWeapp` | `wx.login` 前 await `ensurePrivacyAuthorized('wx.login')` |
| `client/src/adapters/media.ts::chooseVideo` | `chooseMedia` 前 await `ensurePrivacyAuthorized('chooseMedia')` |
| `client/src/pages/analysis/capture.tsx::handleChoose` | 同上（该页未走 adapter，直接补守卫） |

要点：

- 老基础库（< 2.32.3）不支持相关 API，`ensurePrivacyAuthorized` 自动 no-op，行为与 W8 前一致
- 用户拒绝授权时抛 `PrivacyDeniedError`，目前统一走原 catch 分支 toast"选择视频失败"；后续可在 T2/T5 精细化文案
- 登录与拍摄两条核心路径均已守卫

### 12.3 用户协议 / 隐私政策落地

| 产物 | 位置 |
|---|---|
| 用户服务协议 | `client/src/pages/legal/terms.tsx` |
| 隐私政策 | `client/src/pages/legal/privacy.tsx` |
| 首启合规拦截页 | `client/src/pages/consent/index.tsx` |
| 协议版本号 | `client/src/utils/storage.ts::CURRENT_TERMS_VERSION` = `v1.0` |

拦截策略：

- `App.componentDidMount` 读 `storage.hasAgreedCurrentTerms()`；false 则 `reLaunch` 到 consent 页
- 同意后写入本地（版本号 + 时间戳），跳登录页
- 协议文本有实质修订须 bump `CURRENT_TERMS_VERSION`，老用户会在下次启动重新拦截

### 12.4 删除 `scope.userLocation`

`app.config.ts` 原先声明了 `permission.scope.userLocation`（文案"你的位置信息将用于推荐附近球馆"），
但当前版本尚无"附近球馆"功能；保留会触发审核追问"为何需要位置权限"。W8-T1 已删除。
若后续上附近球馆，重新加回并同步隐私政策对应条款。
