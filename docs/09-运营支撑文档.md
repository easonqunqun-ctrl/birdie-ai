## 一、运营管理后台需求

### 1.1 MVP 后台功能清单

MVP 阶段采用轻量级管理后台（可使用 Django Admin / Retool / 自建简易后台），覆盖以下核心运营场景：

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 用户管理 | 用户列表查询 | P0 | 按注册时间、会员类型、活跃度筛选 |
| | 用户详情查看 | P0 | 基础信息 + 分析记录 + 会员状态 + 配额 |
| | 手动调整配额 | P0 | 运营补偿、活动赠送分析次数 |
| | 会员手动开通/延期 | P1 | 客服补偿场景 |
| | 账号注销审核 | P1 | 查看待注销用户，支持手动取消 |
| 内容管理 | 训练动作管理 | P0 | 增删改查 drills 表 |
| | 每日小贴士管理 | P0 | 增删改查 daily_tips 表 |
| | 快捷问题管理 | P1 | 配置 AI 对话快捷问题列表 |
| | 示例视频管理 | P1 | 配置示例体验视频和预设报告 |
| 订单管理 | 订单列表查询 | P0 | 按状态、时间、用户筛选 |
| | 退款处理 | P0 | 审核退款申请，执行退款 |
| 反馈管理 | 反馈列表 | P0 | 查看用户反馈，标记处理状态 |
| | 反馈回复 | P1 | 通过通知向用户回复 |
| 数据看板 | 核心指标 | P0 | 日活/新增/留存/付费 |
| | AI 使用统计 | P1 | 分析次数/对话次数/配额使用率 |
| 系统管理 | 全局配置 | P0 | 会员价格、配额限制、限流阈值 |
| | 版本管理 | P1 | 小程序版本号、强制更新配置 |

### 1.2 后台权限模型

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 全部功能 + 系统配置 |
| 运营人员 | 用户管理 + 内容管理 + 反馈管理 + 数据看板（只读） |
| 客服人员 | 用户详情查看 + 反馈管理 + 手动调整配额 |
| 数据分析师 | 数据看板（只读） |

## 二、数据埋点分析方案

### 2.1 分析工具选型

| 工具 | 用途 | 说明 |
|------|------|------|
| 自建埋点（events） | 核心业务事件 | 通过 POST /v1/events 接口上报 |

### 2.2 核心数据看板指标

**用户增长看板**

| 指标 | 计算方式 | 查看周期 |
|------|----------|----------|
| 日新增用户 | 当日 user_register 事件数 | 日 |
| 日活跃用户（DAU） | 当日有任意事件的去重用户数 | 日 |
| 周活跃用户（WAU） | 7 天内有任意事件的去重用户数 | 周 |
| 月活跃用户（MAU） | 30 天内有任意事件的去重用户数 | 月 |
| 次日留存率 | 注册次日回访用户 / 注册用户 | 日 |
| 7 日留存率 | 注册后第 7 天回访用户 / 注册用户 | 日 |
| 30 日留存率 | 注册后第 30 天回访用户 / 注册用户 | 日 |

**功能使用看板**

| 指标 | 计算方式 | 查看周期 |
|------|----------|----------|
| 日均分析次数 | 当日 analysis_complete 事件数 | 日 |
| 分析完成率 | analysis_complete / analysis_start | 日 |
| 分析平均耗时 | analysis_complete.duration_ms 均值 | 日 |
| 日均对话轮次 | 当日 chat_send 事件数 | 日 |
| 对话平均时长 | 单会话持续时间均值 | 日 |
| 训练打卡率 | 完成任务数 / 总任务数 | 周 |
| 平均连续打卡天数 | users.current_streak_days 均值 | 周 |
| 分享次数 | report_share 事件数 | 日 |

**商业化看板**

| 指标 | 计算方式 | 查看周期 |
|------|----------|----------|
| 付费转化率 | 付费用户 / 注册用户 | 月 |
| 付费墙触发次数 | paywall_trigger 事件数 | 日 |
| 付费墙转化率 | payment_success / paywall_trigger | 日 |
| ARPU | 总收入 / 活跃用户数 | 月 |
| ARPPU | 总收入 / 付费用户数 | 月 |
| 月度 MRR | 当月订阅收入 | 月 |
| 会员续费率 | 续费用户 / 到期用户 | 月 |
| 退款率 | 退款订单数 / 支付订单数 | 月 |

**裂变增长看板**

| 指标 | 计算方式 | 查看周期 |
|------|----------|----------|
| 邀请发出量 | invite_send 事件数 | 日 |
| 邀请转化率 | invite_accept / invite_send | 日 |
| K-Factor（病毒系数） | 邀请用户 × 邀请转化率 / 总用户 | 周 |
| 分享到朋友圈次数 | report_share(channel=moments) | 日 |
| 通过分享码进入用户数 | app_launch(scene=share) | 日 |

### 2.3 数据报表自动化

```
每日报表：
  - 自动生成前一天的核心指标摘要
  - 通过企业微信群机器人推送到运营群
  - 异常指标（环比下降 > 20%）标红告警

每周报表：
  - 周度指标汇总 + 趋势图
  - 重点分析留存和付费转化
  - 每周五下午自动发送

月度报表：
  - 完整数据复盘报告
  - 用户分群分析
  - 营收和增长分析
```

## 三、用户反馈与客服流程

### 3.1 反馈渠道

| 渠道 | 入口 | 处理方式 |
|------|------|----------|
| 小程序内反馈 | 我的 → 意见反馈 | 进入 feedback 表，运营后台处理 |
| 微信客服消息 | 小程序客服按钮 | 接入微信客服系统 |

### 3.2 反馈处理流程

```
用户提交反馈
  ↓
自动分类（关键词匹配）：
  ├── Bug 报告 → 转技术团队
  ├── 功能建议 → 记录产品需求池
  ├── 退款申请 → 转客服 → 退款审核
  ├── 投诉 → 优先处理（24 小时内）
  └── 其他 → 常规处理
  ↓
处理回复（通过小程序通知）
  ↓
标记状态：pending → processing → resolved
```

### 3.3 客服响应标准

| 反馈类型 | 首次响应 | 解决时限 |
|----------|----------|----------|
| 支付问题 | 2 小时内 | 24 小时 |
| Bug 报告 | 4 小时内 | 根据严重等级 |
| 功能建议 | 24 小时内 | 记录并反馈感谢 |
| 投诉 | 2 小时内 | 24 小时 |
| 退款申请 | 4 小时内 | 48 小时 |

### 3.4 常见问题 FAQ

运营需准备以下 FAQ 快速回复模板：

| 问题类型 | FAQ 条目 |
|----------|----------|
| 分析失败 | 拍摄指南链接 + 重新拍摄建议 + 配额已退回说明 |
| 配额用完 | 升级会员引导 + 邀请得配额说明 |
| 支付未到账 | 确认支付状态 + 手动补发权益流程 |
| 取消订阅 | 操作路径指引 + 到期前权益说明 |
| 分析不准确 | 致歉 + 收集反馈 + 模型持续优化说明 |
| 退款请求 | 24 小时内退款政策 + 超期说明 |
| 账号注销 | 注销流程说明 + 数据删除范围 + 7 天冷静期 |

## 四、监控与告警

### 4.1 应用监控（Prometheus + Grafana）

**后端服务监控**

| 监控项 | 告警阈值 | 告警级别 |
|--------|----------|----------|
| API 错误率 | > 1%（5 分钟窗口） | P1 |
| API P99 延迟 | > 2s | P2 |
| CPU 使用率 | > 80% | P2 |
| 内存使用率 | > 85% | P2 |
| 磁盘使用率 | > 90% | P1 |
| 数据库连接池 | 使用率 > 80% | P2 |
| Redis 内存 | > 80% | P2 |
| Celery 队列积压 | > 50 个任务 | P1 |

**业务监控**

| 监控项 | 告警阈值 | 告警级别 |
|--------|----------|----------|
| 分析失败率 | > 10%（1 小时窗口） | P1 |
| 分析平均耗时 | > 60s | P2 |
| AI 对话响应时间 | 首 Token > 5s | P1 |
| 支付成功率 | < 95% | P0 |
| 登录成功率 | < 99% | P0 |
| 视频上传失败率 | > 5% | P2 |

### 4.2 日志管理

**日志收集架构**

```
应用日志 → Tencent Cloud CLS / ELK Stack
  ├── 访问日志（Access Log）
  ├── 错误日志（Error Log）
  ├── 业务日志（Business Log）
  └── AI 推理日志（Inference Log）
```

**日志等级使用规范**

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| ERROR | 需要立即关注的异常 | 支付回调验签失败、数据库连接失败 |
| WARNING | 潜在问题 | 分析超时、限流触发、配额扣减异常 |
| INFO | 关键业务操作 | 用户登录、分析创建、订单支付 |
| DEBUG | 调试信息 | AI 推理中间结果、RAG 检索详情 |

**日志保留策略**

| 日志类型 | 保留时间 |
|----------|----------|
| 错误日志 | 90 天 |
| 业务日志 | 30 天 |
| 访问日志 | 14 天 |
| AI 推理日志 | 7 天 |

### 4.3 告警通知渠道

| 告警级别 | 通知方式 | 响应要求 |
|----------|----------|----------|
| P0（致命） | 电话 + 企业微信 + 短信 | 15 分钟内响应 |
| P1（严重） | 企业微信 + 短信 | 30 分钟内响应 |
| P2（一般） | 企业微信群通知 | 2 小时内响应 |
| P3（轻微） | 仅记录，每日汇总 | 排期处理 |

## 五、灰度发布与上线策略

### 5.1 灰度发布方案

```
阶段 1 - 内测（第 1-2 天）
  ├── 范围：团队内部 + 种子用户（~50 人）
  ├── 目标：发现关键 Bug
  └── 回滚条件：P0/P1 Bug 出现

阶段 2 - 小范围灰度（第 3-5 天）
  ├── 范围：5% 真实用户
  ├── 目标：验证性能和稳定性
  └── 监控：错误率 < 1%，分析成功率 > 85%

阶段 3 - 中等灰度（第 6-7 天）
  ├── 范围：20% 用户
  ├── 目标：观察用户行为数据
  └── 监控：留存指标、付费转化

阶段 4 - 全量发布（第 8 天起）
  ├── 范围：100% 用户
  └── 持续监控 72 小时
```

### 5.2 回滚预案

| 触发条件 | 回滚方式 | 时效 |
|----------|----------|------|
| P0 Bug | 立即回滚上一版本 | 15 分钟内完成 |
| 错误率 > 5% | 停止灰度扩量，评估回滚 | 30 分钟内决策 |
| 支付异常 | 立即关闭支付入口，排查修复 | 即时 |
| 数据不一致 | 停止写入，修复脚本 | 1 小时内 |

## 八、客服：评分与异议话术（ENG-03）

> **真源**：战术模板与产品哲学见 [`docs/20-AI引擎产品力迭代设计.md`](20-AI引擎产品力迭代设计.md) §4.3；**用户可见帮助页**须与 [`client/src/pages/help/score-guide.tsx`](../client/src/pages/help/score-guide.tsx) **同步维护**（措辞或合规调整时两处一起改）。

下列话术可直接用于在线客服、电话回访、社群答疑；可根据渠道适度口语化，**核心含义勿偏离** `docs/20` §4.2「评分是什么 / 不是什么」。

### 8.1 首次出分引导

「这个分数表示：在您**本次拍摄且软件能看清的前提下**，您的动作与领翼golf 当前采用的标准挥杆结构有多接近。**建议您在相近日、相似拍摄条件下多拍几次**，看趋势比盯单次绝对值更有意义。」

### 8.2 顶尖运动员素材低分

「许多职业球员有**强烈的个人技术特点**，在『标准模型』下可能出现**看似偏低的分数**，但不代表其竞技水平。我们正在通过**顶尖样本标定集**持续缩小这种『观感与分数』的割裂。」

### 8.3 重拍 / 机位建议

「本次画面**身体不完整 / 快门角偏低 / 光线过暗**，可能影响关键点检测。**建议侧面全身、光线充足、节拍稳定**后再分析，结果会更稳。」

### 8.4 与教练意见不一致

「AI 从**可见几何与节奏**出发；教练可能还会考虑球路、球场策略等。**若结论冲突，以线下教练为准**；您可以把报告中的维度与教练讨论，我们会持续改进模型。」

### 8.5 争议周更纪要模板（ENG-06）

> **用途**：职业/顶尖样本、高低分争议、线下教练反馈与模型输出的系统性偏差。产品 / 算法 / 运营共用一页纸闭环，结论可回链 [`docs/20` §3.3](../20-AI引擎产品力迭代设计.md#33-教练共识coach-consensus)。

| 字段 | 填写说明 |
|------|----------|
| 周次 / 日期 | 例：2026-W20 |
| 争议来源 | 用户工单 / 内部拨测 / 合作教练反馈 |
| 典型案例摘要 | 1～3 条：杆型、机位、分数区间、主观预期 vs 模型结论 |
| 根因假设 | 标尺 / 关键点 / 视频质量 / 规则边界（择一或多） |
| 处置 | 是否修模型、是否仅话术、是否纳入 ECS 样本待标定 |
| 对外口径 | 是否需要更新帮助页或 `docs/09` §8.x 话术 |
| 跟进 Owner | @ |
| 下周验收 | 可测标准（截图 / 单测 / ECS 回归） |
