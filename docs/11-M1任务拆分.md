## 推进顺序建议

```
T1 (后端接口) ──► T2 (登录打通) ──► T3 (引导)
                                      └──► T4 (我的最小版，可与 T3 并行)
                                               └──► T5 (文档收尾，M1 收官)
```

每个 Txx 一个 PR；PR 标题示例：

```
feat(backend): M1-T1 wechat-login + /users/me 打通
feat(client): M1-T2 登录页对接真实接口
feat(client): M1-T3 新用户三步引导
feat(client): M1-T4 我的页与档案编辑最小版
docs: M1-T5 同步 API/DB/MVP 验收清单
```
