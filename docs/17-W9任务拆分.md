## 二、做与不做

### 做（W9 交付）

| 场景 | 交付 |
|---|---|
| 真实微信登录 | `WECHAT_MOCK_LOGIN=false`，真 AppID/Secret 下真机登录通过 |
| 真实 LLM | 关闭 mock LLM，DeepSeek 流式对话在真机稳定返回 |
| 真实支付 | wechatpayv3 下单、prepay、回调验签、会员激活、订单状态机 |
| 正式对象存储 | MinIO 切腾讯云 COS，视频 / 关键帧 / 缩略图走 COS + CDN |
| 正式部署 | TKE 或腾讯云 CVM 生产栈，HTTPS、日志、备份、健康检查 |
| 合法域名 | 小程序后台配置 request/upload/download/socket 合法域名 |
| 审核材料 | 类目、隐私指引、服务协议、测试账号、审核说明 |
| 灰度上线 | 1% → 10% → 50% → 100%，观察错误率 / 支付转化 / 分析成功率 |
| W8 补测 | Android 真机 + onboarding / 邀请 / mock_pay / 违规素材 / SDK 低版本补测 |

### 不做（W9 之后）

| 项 | 原因 |
|---|---|
| 运营后台 CMS | 先用 SQL / 脚本运营，待用户规模起来再做 |
| GPU 化 AI 引擎 | CPU 版 MediaPipe 先撑 MVP，按压测再扩 |
| 家庭版会员 | 先上线 monthly/yearly，家庭版延后 |
| 订阅消息精细运营 | 审核上线后再申请模板并迭代 |

## 三、任务拆分

### T1 · 上线资源与账号准备（P0）

- 注册 / 确认正式域名与子域名规划：`api`、`cdn`、`video`、`www`
- 完成 ICP 备案、小程序备案、微信支付商户号申请
- 准备微信支付 APIv3 key、商户证书、回调地址
- 准备 DeepSeek API key 与最小预算充值
- 输出 `docs/release-notes/W9-launch-resource-checklist.md`

### T2 · 真实微信登录 + 真实 LLM 切换（P0）

- 测试环境设置 `WECHAT_MOCK_LOGIN=false`
- 验证 `jscode2session` 真机登录，`users.wechat_openid` 非 `mock_*`
- 设置 `LLM_PROVIDER=deepseek`，验证 SSE 流式、超时、错误兜底
- 补 W8-T6 的 Step 2 / Step 6 真实通路走查

### T3 · 真实微信支付（P0）

- 接入 wechatpayv3 SDK 或轻量签名封装
- `/v1/payments/orders` 生成 prepay 参数
- 小程序端调 `requestPayment`
- `/v1/payments/wechat/notify` 验签、幂等更新订单、激活会员（实现路径含 `/wechat/`；配置 **`WECHAT_PAY_NOTIFY_URL`** 须与之完全一致）
- 订单超时关闭 / 重复回调 / 退款占位
- 后端测试覆盖：下单、回调验签、幂等、会员激活

### T4 · COS / CDN 存储切换（P0）

- 新建 COS bucket：视频、关键帧、缩略图分前缀
- 后端 storage adapter 从 MinIO 切 COS（保持接口不变）
- CDN 绑定 `video.*` / `cdn.*`
- 小程序合法域名配置 upload/download
- 历史测试数据不迁移，W9 正式环境从空库启动

### T5 · 正式部署与安全加固（P0）

- 选择部署形态：
  - MVP 推荐：2 台 CVM + TencentDB PostgreSQL + Redis + COS/CDN
  - 规模化再切：TKE + Ingress + HPA
- 配置 HTTPS、环境变量、日志、备份、健康检查、告警
- 生产 `QUOTA_MODE=strict`、`PAYMENT_ENABLED=true`
- 关闭 debug badge 与测试 tunnel 依赖

### T6 · 小程序提审准备（P0）

- 后台配置合法域名、业务域名、隐私指引、服务类目
- 准备审核账号、审核说明、测试视频、支付测试说明
- 检查协议页 / 隐私页 / 账号注销页
- 扫描敏感词、调试浮层、mock 文案、测试域名

### T7 · 灰度发布与回滚（P0）

- 提交审核，通过后先 1% 灰度
- 观察核心指标：
  - 登录成功率
  - 分析成功率
  - 平均分析耗时
  - SSE 对话成功率
  - 支付成功率
  - error_report 数量
- 准备回滚策略：小程序版本回退、后端镜像回退、支付入口 flag 关闭

### T8 · W8 遗留补测与产品小修（P1）

- Android 真机完整走查
- Onboarding 新用户路径补测
- 邀请码复制 / 口令文案补测
- mock_pay / 会员状态补测
- 关键图产品策略确认：
  - 诊断问题数量保持真实动态，还是强制凑 Top 5
  - 点关键图是看大图、跳静帧，还是自动播放
