## 三、模块投入与人力配比（建议）

> 与 [`docs/21` §3.2](21-二期产品需求规划.md#32-二期总投入大致分布建议) 投入分布一致；下表把投入折算成"等效工程人周"作为排期校验。
>
> **占比是区间**：上限相加 > 100% / 下限相加 < 100% 是正常的，因为每个模块的实际投入会按 RISK-01 ~ RISK-09 触发情况在区间内浮动；评审定档时再压成具体数值。

| 模块 | 投入占比（区间） | 等效人周（按 12 人 × 26 周 = 312 人周估算） | 主要角色 |
|------|---------|----------------------------------|--------|
| M7 引擎 V2 | 40-50% | 125-156 人周 | 算法 ×2 + AI 工程 ×1.5 + 产品 ×0.5 |
| M8 教练工作台 | 12-15% | 38-47 人周 | 后端 ×1.5 + 前端 ×1.5 + 产品 ×0.5 |
| M10 短杆 / 推杆 | 8-12% | 25-37 人周 | 算法 ×0.5 + 前端 ×0.5 + 教研 ×0.3 |
| M11 课程体系 | 7-10% | 22-31 人周 | 教研 ×1 + 前端 ×1 |
| M12 职业球手对比 | 7-10% | 22-31 人周 | 内容运营 ×1 + 前端 ×1 + 算法 ×0.3 |
| M9 画像 2.0 | 5-10% | 16-31 人周 | 后端 ×1 + 前端 ×0.5 |
| M14 RN 发版 | 5-10% | 16-31 人周 | RN 工程 ×1 |
| M13 球友约球 | 5% | 16 人周 | 后端 ×0.5 + 前端 ×0.5 + 风控 / 法务 ×0.3 |
| **下限合计** | 89% | ~280 人周 | 保留 ~10% 为非模块工作（CI / 工具 / 文档同步 / on-call / 回归）|
| **上限合计** | 122% | ~380 人周 | 不可能同时按上限走；评审时按 RISK 兜底决定 |

> 若团队规模 < 10 人，建议先砍 M14 RN 发版（推到 Phase 2.6 单独 sprint）、M10 收紧到推杆 mode only、M12 收紧到不含追平演化动画。

## 九、UI / 交互一致性约束（二期延续一期）

> **核心原则**：二期所有新页面 / 新组件 **不引入新 UI 库**、**不重做信息架构**、**不脱离白皮书 §7.2 视觉规范**。

| 维度 | 约束 |
|------|------|
| **颜色** | 一律走 `client/src/app.scss` 既有 CSS 变量；新模块如需配色，**先**到白皮书 §7.2 找最接近语义，不够再补；**严禁**业务页面硬编码品牌 HEX |
| **字体 / 字号** | 与一期 Profile / Report / Training 三大主页面对齐 |
| **组件复用** | 优先复用一期已有组件：`DrillCard`、`VideoCard`、`AnalysisCard`、`ProgressLineChart`、`PracticeCalendar`、`EnvBadge` 等；新组件需走代码 review 双签（前端 + 产品） |
| **页面框架** | 沿用一期 tab 框架（首页 / 训练 / 报告 / 教练 / 我的）；二期新模块 **不新增 tab**，按子页面接入既有 tab；M8 教练工作台通过"我的→切换教练身份"进入；M12 资源库通过"教练对话"上方 banner 或独立子页面进入；M13 约球通过"训练→找球友"按钮进入 |
| **动效** | 沿用一期既有 Taro `animation` API + CSS transition；不引入 lottie / framer-motion 等三方动画库 |
| **交互模式** | 沿用一期：垂直滚动 / 点按 / 长按；**严禁** 双指缩放 / 复杂手势（RN 兼容性差） |
| **错误态 / 加载态 / 空态** | 复用一期 `ErrorState` / `LoadingState` / `EmptyState`（若一期没有抽象则在 Phase 2.0 抽象一套） |
| **图标** | 复用 W8 TabBar 图标体系；新增图标必须按白皮书 §7.2 金色 + 冷靛蓝双色体系 |
| **跨端守则** | AGENTS.md §4 全部仍然适用：平台差异 API 走 `src/adapters/*`；样式 flex 为主；新页面三件套 `.tsx` / `.scss` / `.config.ts` |

**校验机制**：

- Code Review 强制项：PR 模板加 `[ ] 已遵循 docs/22 §九 UI / 交互一致性约束`
- Lint 守门（建议二期 Phase 2.0 落地）：脚本 grep 业务页面 SCSS 中的 `#xxx`，命中品牌色硬编码即 fail
- 视觉走查：Phase 2.1 / 2.2 / 2.3 / 2.4 各 milestone 前由产品 + 设计联走每个新页面
