## 二、核心能力 · 五件事（近 6 周必关）

> 验收真源：[`experience-version-smoke-runbook.md`](./experience-version-smoke-runbook.md) **「线上待验 P0」** + 下方补充项。

### ✅ 1. 正式版主链路可信（P0）

**目标**：用户第一次分析不「白跑一趟」。

| 验收项 | 通过标准 |
|--------|----------|
| 全挥杆 | capture → params → 等待 → **报告有 issue / drill / 弱项**（差样本不能「没有明显问题」） |
| 多挥 | 真多挥才进 select-swing；单段不进 |
| 画像 | **我的 → 我的画像** 无 500 |
| 合规 | AI 教练 Tab **未登录可浏览**（1.2.34 已发，正式版须再验） |

**负责人动作**：正式版真机勾 smoke runbook **「线上待验 P0」** → 全绿记日期。

---

### ✅ 2. 引擎与报告「看得懂」（P0+）

**目标**：报告像教练说话，不像 debug 输出。

| 验收项 | 通过标准 |
|--------|----------|
| 低分样本 | 有明确诊断与建议（非空 issue 列表） |
| 机位 / 可信度 | 侧面 absurd 旋转类文案已止血；Trust/机位说明可见 |
| 等待体验 | 阶段推进合理；失败可重试、配额不误扣 |

**锚点**：近期 1.2.33+ 引擎/报告修复；**必须在正式版再验一轮**。

---

### ✅ 3. 训练闭环走得通（P1）

**目标**：分析完知道「练什么」。

| 验收项 | 通过标准 |
|--------|----------|
| issue → drill | 报告弱项能跳到训练/对应 drill |
| 训练 Tab | 计划/任务可读；打卡或完成态不崩 |
| 短杆（若开 flag） | 推杆/切杆至少 smoke 一条 E2E |

**暂不要求**：drill 专属视频全齐、M12 演化动画。

---

### ✅ 4. AI 教练 + 会员页不挡路（P1）

**目标**：测试期用户问得动、不被支付墙吓跑。

| 验收项 | 通过标准 |
|--------|----------|
| 教练 | 登录后对话可用；免费期 **不因 5 轮/日卡死**（见 §四 配额） |
| 会员页 | 展示 **「测试期免费至 7/30」** 说明；支付入口可保留但非主 CTA |
| 首页 | 不因「本月 3 次用完」弹窗阻断（配额放宽期间） |

---

### ✅ 5. 传播「只开一条线」（7 月中旬后，P2）

**前置**：§二 1–4 至少 P0 全绿。

| 只做一件事 | 说明 |
|------------|------|
| **分析完成 → 分享海报** | 已有 [`poster.tsx`](../../client/src/pages/analysis/poster.tsx) + wxacode |
| 种子渠道 | 2–3 个球友群 / 1 个合作教练带学员；**不铺抖音/投放** |
| 观测 | 分享点击 → 新注册 → 首次分析完成率 |

**不做**：球场测距、新 Tab、大范围 Phase2 功能宣传。

## 四、测试期免费（至 2026-07-30）· 执行说明

### 4.1 产品承诺（对外话术）

> **领翼 golf 公测期：至 2026 年 7 月 30 日，分析次数与 AI 对话暂不限制，欢迎体验反馈。**

建议展示位置：

- 首页顶部轻提示 / 关于页
- 会员页副标题（「公测免费中，8 月起恢复会员体系」——8 月细则待定）

### 4.2 后端：配额怎么放（现有能力）

| 机制 | 作用 | 现状 |
|------|------|------|
| `QUOTA_MODE=unlimited` | 免费用户 **分析月配额 + 对话日配额** 均视为无限 | [`env.prod.deployment.template`](../../infra/deploy/env.prod.deployment.template) 已默认 `unlimited`；**须核对 CVM `.env.local` 实际值** |
| `QUOTA_MODE=strict` | 免费 3 次分析/月、5 轮对话/日 | 若 CVM 为 strict，测试期应改为 `unlimited` |

**操作（运维）**：

1. SSH CVM 查看 `QUOTA_MODE`
2. 若非 `unlimited`：`QUOTA_MODE=unlimited` → `make release-cvm`（或等价重建 backend）
3. **日历提醒 2026-07-31**：改回 `strict` 或启用 §五 正式策略

**注意**：`production_guard` 在 `APP_ENV=prod` 时会 **拒绝** `QUOTA_MODE=unlimited`。当前 API 若为 `APP_ENV=staging`（模板即 staging），则无此拦；若将来切 `prod`，需加 **按日期促销 env** 或临时 guard 例外（见 §4.4）。

### 4.3 仍未被 `QUOTA_MODE` 解除的限制（需知悉）

| 限制 | 位置 | 测试期建议 |
|------|------|------------|
| 历史报告仅展示 **3 份** | `GET /analyses` paywall | 用户第 4 份起列表截断 — **易误解为「丢报告」**；种子用户量小可暂忍，或排期短期关闭 paywall |
| 训练 **进步曲线** 非会员锁定 | `training/index` 客户端 | 可保留锁定 UI，或测试期文案改为「公测即将开放」 |
| 部分课程 `is_member_only` | 课程 Tab | 测试期可暂不推课程 Tab 获客 |
| 微信支付 | 已接真支付 | 会员页仍可展示套餐，**主运营话术强调免费**；避免用户误付（会员页加公测说明） |

### 4.4 8 月前可选工程项（非必须，体验更完整）

若免费期用户反馈「还是被挡」，按优先级：

1. **PROMO_FREE_UNTIL=2026-07-30** env + 后端在日期前视同会员配额（比长期 `unlimited` 更安全）
2. 测试期内 **关闭** `AnalysisListPaywall`（或 `capped_to` 提高到 99）
3. 客户端首页 **隐藏**「本月次数已用完」弹窗（当 `quota.remaining < 0` 且促销中）

> 当前文档 **不强制改代码**；先 `QUOTA_MODE=unlimited` + 文案即可启动。若你确认要改，单独立项 **feat/promo-free-until-0730**。

## 六、传播 · 最小 playbook（7 月中旬启用）

**前提**：§二 **事项 1 P0 全绿**。

1. **素材**：报告页 → 「生成分享海报」→ 球友群 / 朋友圈（带小程序码）
2. **话术**：「免费帮你看挥杆问题，公测到 7 月 30 日」
3. **邀请**（若已开）：沿用现有邀请赠次逻辑；与 `unlimited` 叠加时 **邀请动力弱**，测试期以 **体验口碑** 为主
4. **收集反馈**：`pages/profile/feedback` + 固定微信群；每周汇总 **Top3 报告离谱 case**

**不投**：球场 POI、测距、应用商店 ASO、付费投放。
