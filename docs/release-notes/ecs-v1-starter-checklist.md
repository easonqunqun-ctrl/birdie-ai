# ECS v1 启动清单（工程 / 运营）

> **定位**：顶尖与职业代表样本标定集（Elite Calibration Set）在 **ENG-04** 中的**启动与协作清单**。
> **真源**：[`docs/20-AI引擎产品力迭代设计.md` §5](../20-AI引擎产品力迭代设计.md#五世界顶尖样本标定集elite-calibration-set-ecs)。
> **更新**：v1.1.2（2026-05-22）从「5 段提示」扩为可执行的角色矩阵 / 评审节奏 / 漂移门禁。

本清单**不等于** ECS 数据集已齐备或 `ai_engine` 已接入自动化回归；用于跨角色对齐「授权—标签—存储—评审—门禁」的最小流程。

---

## 0. 角色矩阵

| 角色 | 主要职责 | 关键产出 |
|------|----------|----------|
| **产品** | 拍板「进 ECS 的样本类型」、争议结论、漂移阈值 | §4 评审节奏纪要、§5 阈值定义 |
| **运营** | 联系样本来源、授权回收归档、争议样例触发 | §1 授权台账、§4 例会议程 |
| **算法 / 工程** | ECS 接入 `ai_engine` 回归脚本、漂移指标实现、CI 门禁 | §4 快照对比、§5 门禁代码 |
| **客服** | 把用户争议样例回流为 ECS 候选 | 周表「争议样例 → 候选 ID」 |

任一角色缺位 ECS 都跑不起来；新人加入第一周必须读完本文件 + [`docs/20` §5](../20-AI引擎产品力迭代设计.md#五世界顶尖样本标定集elite-calibration-set-ecs)。

---

## 1. 授权与合规

- [ ] 每条素材具备可分析的书面授权范围（线上产品展示 / 内部分析 / 第三方不可转授）。
- [ ] 存证：授权文件版本、有效期、联系人归档路径（内部 wiki 或对象存储前缀）。
- [ ] 授权台账固定字段：`ecs_clip_id` · `来源` · `授权类型` · `有效期` · `联系人` · `归档路径`；缺一项不入库。
- [ ] 涉及未成年人 / 残障人士 / 教学机构 logo：单独列「敏感清单」，**不进 v1 ECS**，留到合规审过再用。

---

## 2. 元数据与标签

- [ ] 统一 ID 规则（与内部 `analysis_id` 或 `ecs_clip_id` 映射表）。
- [ ] 必填标签：杆型、机位、性别 / 身高档位（若公开）、**预期分位档**或「teaching 标杆 / 争议样例」分类。
- [ ] 争议样例须有 **一句话** 产品结论（为何进 ECS）。
- [ ] 推荐分位档（v1）：`P95+` teaching 标杆 · `P80～P95` 顶尖业余 · `P50～P70` 进阶参照 · `争议` 单独分类，不参与百分位计算。
- [ ] v1 目标体量：**teaching 标杆 ≥ 20 条 · 顶尖业余 ≥ 30 条 · 争议 ≥ 10 条**；不到这个量不开漂移门禁。

---

## 3. 存储与访问

- [ ] 对象存储前缀与生产桶隔离（仅内网或签名访问）。
- [ ] 与 `ai_engine` mock/real 配置对齐的环境变量或配置项清单（不在本文写死密钥）。
- [ ] 推荐前缀：`ecs/v1/<class>/<ecs_clip_id>.mp4`（`class ∈ {teaching, top_amateur, dispute}`），与 `swing_analyses` 主链路桶**分桶**避免 GC 误删。
- [ ] 读权限白名单：算法工程师 + 产品；客服无直接读权限，由运营按周批量导出脱敏关键帧。

---

## 4. 评审与回归（v1 节奏锁定）

- [ ] **评审例会**：**双周**周三 19:00，60 分钟；纪要落档（模板见 [`docs/09` §8.5](../09-运营支撑文档.md#85-争议周更纪要模板eng-06)）。
- [ ] 例会议程固定 4 段：① 新增样本快速过 · ② 争议样例结论 · ③ 漂移指标周看板 · ④ 下双周 backlog。
- [ ] **快照对比门禁**：每次 `ai_engine` 主分支合并前，对 ECS 子集跑回归，输出每条样本的 6 维 / overall 与上一基线的 delta。
- [ ] **门禁口径**（v1，可在评审例会调整）：
  - 单条 overall delta > **±5** → 标黄，需算法说明
  - 单维 delta > **±8** → 标黄
  - teaching 标杆类样本 overall **绝对值 < 80** → 标红，必须回滚或开 exception
  - 50% 以上样本同方向漂移 > **±3** → 标红，禁止合并

## 5. 与产品文档的同步

- [ ] 字段或接口扩展先改 `docs/02` / `docs/03`，再改代码（[cite AGENTS.md](../../AGENTS.md)）。
- [ ] ECS 相关脚本与数据规格变化先更新 [`docs/20` §5](../20-AI引擎产品力迭代设计.md#五世界顶尖样本标定集elite-calibration-set-ecs) 与本文件，再合并实现。
- [ ] 客服话术涉及「报告分数与教练评估出入」时，必须显式引用 [`docs/09` §8.4](../09-运营支撑文档.md) 与 ECS 评审纪要锚点，避免运营即兴回复。

---

## 6. v1 启动检查（DoD）

正式宣布「ECS v1 启动」需要同时满足：

- [ ] §1 ～ §3 全部前置项可在台账中 grep 到
- [ ] §2 标签字段全部进入 ECS 元数据表
- [ ] §4 第一次例会已开（哪怕只有 5 条样本）
- [ ] §4 门禁脚本至少能在 CI 跑出一份对比报表（即使全部 PASS）

未满足时 ECS 仍可**只读**，但不得对外宣称「已用于回归」。
