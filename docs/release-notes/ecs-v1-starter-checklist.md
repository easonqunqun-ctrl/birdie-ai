# ECS v1 启动清单（工程 / 运营）

> **定位**：顶尖与职业代表样本标定集（Elite Calibration Set）在 **ENG-04** 中的**启动与协作清单**。  
> **真源**：[`docs/20-AI引擎产品力迭代设计.md` §5](../20-AI引擎产品力迭代设计.md#五世界顶尖样本标定集elite-calibration-set-ecs)。

本清单**不等于** ECS 数据集已齐备或 `ai_engine` 已接入自动化回归；用于跨角色对齐「授权—标签—存储—评审」的最小流程。

## 1. 授权与合规

- [ ] 每条素材具备可分析的书面授权范围（线上产品展示 / 内部分析 / 第三方不可转授）。
- [ ] 存证：授权文件版本、有效期、联系人归档路径（内部 wiki 或对象存储前缀）。

## 2. 元数据与标签

- [ ] 统一 ID 规则（与内部 `analysis_id` 或 `ecs_clip_id` 映射表）。
- [ ] 必填标签：杆型、机位、性别/身高档位（若公开）、**预期分位档**或「 teaching 标杆 / 争议样例」分类。
- [ ] 争议样例须有 **一句话** 产品结论（为何进 ECS）。

## 3. 存储与访问

- [ ] 对象存储前缀与生产桶隔离（仅内网或签名访问）。
- [ ] 与 `ai_engine` mock/real 配置对齐的环境变量或配置项清单（不在本文写死密钥）。

## 4. 评审与回归（后续里程碑）

- [ ] 评审例会节奏（双周 / 月）与纪要落档（模板见 [`docs/09` §8.5](../09-运营支撑文档.md#85-争议周更纪要模板eng-06)）。
- [ ] `ai_engine` 在 ECS 子集上的**快照对比**门禁：漂移阈值由产品与算法共担，写入 CI 或发布前 checklist。

## 5. 与产品文档的同步

- [ ] 字段或接口扩展先改 `docs/02` / `docs/03`，再改代码（[cite AGENTS.md](../../AGENTS.md)）。
