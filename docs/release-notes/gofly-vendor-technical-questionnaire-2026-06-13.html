问题一：是否存在「无硬件依赖」的数据接入方式？

文档中 getNearbyCourseList、setCurrentCourse 等接口均要求 guid + deviceType，且设备授权流程依赖蓝牙 [GUI]、 球场升级依赖 [CLC] 指令下发。请确认是否提供以下任一（或多种） 不绑定硬件的接入形态：

  • 服务端 HTTP/HTTPS API：由我方后端持密钥调用，小程序只调自有 API
  • 纯手机 App 授权：仅需 deviceType + 应用级密钥，无需蓝牙 guid
  • 球场数据离线包：批量提供 json + 洞图/布局 + crc32，我方自建缓存与升级
  • 若仅支持硬件路径，是否可为我方单独开通「软件-only」授权档位

若答案为「仅硬件 + 原生 SDK」，请明确说明，我方将评估换源或改交付形态。

问题二：微信小程序场景是否在授权与技术上可行？

我方主载体为微信小程序（Taro），非 Flutter 独立 App。请确认：

  • 商业授权是否允许用于微信小程序（含体验版 / 正式版）
  • 是否提供非原生 SDK 接口（REST / WebSocket / 数据包），或是否允许我方在自有后端调用贵方服务后转给小程序
  • 密钥 / 配额绑定维度：按应用、按日活用户、还是仍按硬件 guid
  • 球场洞图、障碍/果岭多边形等，能否以 URL 或 json 形式输出（便于我方用微信 map + addGroundOverlay 渲染）

我方技术栈约束：客户端为 Taro 3 + 微信基础库；定位使用 wx.getLocation（GCJ-02）； 无法嵌入 Flutter / Android / iOS 原生 SDK。
