# MVP 验收 Runbook：O-01 / O-04 与并行工程健康（par）

> 与 [`docs/01-MVP功能需求规格说明书.md`](../01-MVP功能需求规格说明书.md) 性能与大文件上传条目对齐；**以真机数据为准**，记入运维/版本说明即可，不必改业务代码除非发现明确瓶颈。

## O-01 真机性能（小程序）

| 步骤 | 记录项 |
|------|--------|
| 设备 | 机型、微信版本、基础库版本 |
| 冷启动 | 首页可交互耗时（秒） |
| 报告页 | 含视频首帧 + 雷达图可交互耗时 |
| 重复进入 | 第二次进入同一报告是否明显快于首次（缓存/预加载） |

## O-04 大文件上传

| 步骤 | 记录项 |
|------|--------|
| 网络 | Wi-Fi / 4G / 5G |
| 文件 | 时长（秒）、大小（MB）、分辨率 |
| 结果 | 是否一次成功；失败时 HTTP 状态与是否触发凭证刷新重试 |
| 网关 | API 与直传域名是否均在小程序「合法域名」内 |

## par（Partial）收口建议

- **Toast / 错误文案**：弱网场景下 `describeIntermittentRequestFailure` 与业务码是否可区分（避免一律「网络错误」）。
- **pytest**：`make backend-test` 在 CI 通过；关键路径含支付 mock、分析生命周期、订阅消息 noop。

## W6 交叉

真实引擎预发：`AI_ENGINE_MOCK_MODE=false`，短视频 fixture 跑通后，在此 Runbook 补一条「真实 pipeline 报告生成」截图或日志锚点（时间、request_id）。
