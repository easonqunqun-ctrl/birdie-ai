## 三、模块设计

### 3.1 改造 + 新增

| 模块 | 路径 | 工程量 |
| --- | --- | --- |
| 200 条数据冷启 | 运营 + Excel 批量导入 | 1 PW |
| API | GET /v1/meetup/venues + POST /v1/meetup/venues + report | 0.5 PW |
| 后台 venues 模块 | `admin/venues/` | 0.7 PW |
| UGC + 审核工作流 | service 层 | 0.5 PW |
| 单测 | tests | 0.3 PW |

**合计：~3 PW**

### 3.2 10 大城市冷启分配

| 城市 | 数量 |
| --- | --- |
| 北京 / 上海 / 深圳 / 广州 | 各 ≥25 |
| 杭州 / 成都 / 武汉 / 西安 / 苏州 / 重庆 | 各 ≥20 |
| **共计** | **≥230** |

每场地：name + venue_type + city + address 必填。

### 3.3 UGC + 审核

```
用户/教练 POST /v1/meetup/venues → source='ugc' status='active' (即时可用)
                                  → 加入运营队列 7 天内复核
                                  → 复核通过：source='verified'
                                  → 复核未过：status='flagged' → P2-M9-05 不展示
```

### 3.4 举报

```
POST /v1/meetup/venues/{id}/report
  → 写 venue_reports 表（新表 v0.2）
  → 累计 ≥3 次报告自动 status='flagged'
  → 运营人工 review
```
