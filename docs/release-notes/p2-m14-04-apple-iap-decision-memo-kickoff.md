## 三、模块设计（决策备忘）

### 3.1 决策

**iOS App 内付费入口跳转小程序订阅引导。不做 Apple IAP。**

理由：
- Apple 30% 抽成，二期会员客单价低（¥38/月），抽成大幅压缩利润
- 小程序付费已稳定运行，复用即可
- 集中付费链路便于运营 + 风控

### 3.2 跳转话术（避免 3.1.1 拒）

```
[会员订阅]
本服务暂未开放 App 内订阅。
您可以在 [我的 - 微信小程序] 中查看会员权益。
```

**严禁**写法：
- ❌ "请前往微信小程序购买"（主动引导外部支付）
- ❌ "点此跳转支付"（含跳转按钮 + 价格 + URL）
- ❌ "App Store 外更优惠"（价格诱导）

**法务 + 产品 + 苹果审核团队三方签字** 后方可上架。

### 3.3 反向验证

- 通过 App Store 审核（不被拒 3.1.1）
- 用户 iOS 端付费转化率不低于小程序端 70%（运营 KPI）

### 3.4 未来评估窗口

| 触发 | 评估动作 |
| --- | --- |
| Apple 抽成下调 ≤15% | 重新评估 IAP |
| 苹果政策针对中国 App 调整 | 重新评估 |
| 我们国际化 | 重新评估（海外用户必须 IAP） |
