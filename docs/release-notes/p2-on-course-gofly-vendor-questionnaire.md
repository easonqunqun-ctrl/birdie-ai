## 一、文档目的

向 Gofly（高飞通信）确认球场地图数据在**无硬件**前提下，能否支撑领翼 golf 下场 MVP（测距 + 记分），并明确授权形态、数据字段与试用条件，以便 1 周内决策是否立项 **M14 下场模块**。

### 边界（本文档不涉及）

- 不讨论领翼侧 AI 策略文案 / GIR·FIR 自动判定实现细节（应用层自研）
- 不承诺采购或签约；仅技术可行性确认
- 不修改 `docs/21` / `docs/23`（待 Gofly 回复后再写 PRD）

---

## 三、问题一：是否存在「无硬件依赖」的数据接入方式？

文档中 `getNearbyCourseList`、`setCurrentCourse` 等接口均要求 `guid + deviceType`，设备授权依赖蓝牙 `[GUI]`，球场升级依赖 `[CLC]` 指令下发。

请确认是否提供以下任一（或多种）**不绑定硬件**的接入形态：

| 子项 | 请确认（是 / 否 / 部分） | 说明 |
|------|-------------------------|------|
| 1.1 | | **服务端 HTTP/HTTPS API**：我方后端持密钥，小程序只调自有 API |
| 1.2 | | **纯手机授权**：仅需 `deviceType` + 应用级密钥，**无需**蓝牙 `guid` |
| 1.3 | | **球场数据离线包**：批量提供 json + 洞图/布局 + crc32，我方自建缓存与升级 |
| 1.4 | | 若仅支持硬件路径，是否可单独开通 **「软件-only」** 授权档位 |

**若答案为「仅硬件 + 原生 SDK」**，请明确说明；我方将评估换源或改交付形态。

---

## 四、问题二：微信小程序场景是否在授权与技术上可行？

我方主载体为 **微信小程序（Taro）**，无法嵌入 Flutter / Android / iOS 原生 SDK。

| 子项 | 请确认 | 说明 |
|------|--------|------|
| 2.1 | | 商业授权是否允许用于 **微信小程序**（体验版 / 正式版） |
| 2.2 | | 是否提供 **非原生 SDK** 接口（REST / WebSocket / 数据包），或允许我方 **后端代理** 后转给小程序 |
| 2.3 | | 密钥 / 配额绑定维度：按 **应用**、按 **日活**、还是仍按 **硬件 guid** |
| 2.4 | | 洞图、障碍/果岭多边形能否以 **URL 或 json** 输出（便于 `map` + `addGroundOverlay`） |

**我方技术栈约束（供评估）**

- 客户端：Taro 3 + 微信基础库
- 定位：`wx.getLocation`（GCJ-02）；下场可能扩展 `startLocationUpdate`（前台）
- 已有能力：[`client/src/adapters/location.ts`](../../client/src/adapters/location.ts)（M9-05 常去球馆定位）

## 五、问题三：数据覆盖、能力与我方 MVP 字段是否对齐？

我方首期 MVP（无硬件）需同时支持 **测距** 与 **记分**。请确认：

### 5.1 覆盖范围（请填写）

| 项 | 请提供 |
|----|--------|
| 中国大陆可授权球场总数 | 约 ____ 座 |
| 数据更新频率 | 如：每月 / 按 crc32 增量 |
| 废弃球场策略 | 是否提供 §7.4 `getNoUserCourse` 同类能力 |
| 首期试点 | 我方指定 1–3 家球场开通验证是否支持 |

### 5.2 单场 / 单洞必备数据（请勾选已支持）

- [ ] 球场 ID、名称、经纬度
- [ ] 18 洞标准杆（`getHolePars`）
- [ ] 各洞发球台坐标 + 官方洞距（`getTeeDistance` / `getCurrentTeeCoordinate`）
- [ ] 洞图图片或 bounds（地图叠层）
- [ ] 果岭多边形 + pin 点 + 前/中/后距离（`getGreenDistances`）
- [ ] 障碍/沙坑多边形（`getHazardCoordinates`）
- [ ] 球道参考点（`getHazardFairways`）— FIR 辅助
- [ ] GPS 自动切洞（`calculateDistanceAndSwitchHole`，15 码规则）— 可由我方实现，需 tee 坐标

### 5.3 明确不支持的能力（请列出）

例如：实时策略文案、记分卡、GIR/FIR 自动判定等——我方理解由应用层自研，仅需贵方 **地理与洞图数据**。

### 5.4 报价与试用（请填写）

| 项 | 请提供 |
|----|--------|
| 「软件-only / 服务端 API」报价模型 | 年费 / 按球场数 / 按调用量等 |
| 试用 | 是否提供试用密钥 + 1–3 个球场样本（建议 2 周联调） |
| 对接人 | 姓名 / 微信 / 邮箱 |

## 七、我方首期产品范围（供贵方评估授权场景）

| 功能 | 用户价值 | 对 Gofly 的依赖 |
|------|----------|-----------------|
| **下场测距** | 当前 GPS → 果岭前/中/后；可选手动打点测距 | 洞图 + 坐标 + 距离或原始多边形 |
| **记分** | 逐洞杆数、推杆；赛后 GIR/FIR/推杆汇总 | 标准杆；GIR/FIR 可选依赖果岭/球道几何 |
| **AI 选杆（简版）** | 剩余码数 + 个人杆距表荐杆 | 仅距离数据；不依赖贵方 AI |
| **暂不包含** | — | 蓝牙手表、硬件授权、`[CLC]` 下发 |
