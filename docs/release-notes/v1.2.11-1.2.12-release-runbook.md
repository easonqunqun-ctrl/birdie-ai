# v1.2.11–1.2.12 合并发版 Runbook

> **范围**：v1.2.11（Q-B1 到期弹窗 + par-T1 pytest）+ v1.2.12（Batch-D 引擎质量门禁 + `/precheck`）  
> **须同时发**：**CVM ai_engine + backend**（Celery 调 `/precheck`）+ **微信小程序体验版**

---

## 1. 发版前自检（本地 / CI）

```bash
# 客户端
cd client && pnpm type-check && pnpm exec jest --no-coverage

# 后端（Docker 或 uv 环境）
make backend-test

# ai_engine（Docker 或 uv 环境）
cd ai_engine && uv run pytest tests/test_preprocess.py tests/test_pose.py -q
```

---

## 2. CVM 发版顺序

```bash
# 1) 拉代码
cd ~/lingniao-golf && git pull --ff-only

# 2) 重建 ai_engine + backend + worker（须含新 /precheck 路由）
bash infra/deploy/cvm-rebuild-backend.sh   # 或等价 compose rebuild

# 3) 健康探针
curl -sS "https://api.birdieai.cn/v1/health" | python3 -m json.tool | head -40
# 期望：services.ai_engine=ok，ai_engine.mock_mode=false

# 4) ai_engine precheck 冒烟（任选一条已上传视频的 URL）
curl -sS -X POST "http://127.0.0.1:8001/precheck" \
  -H "Content-Type: application/json" \
  -d '{"analysis_id":"smoke_precheck","video_url":"..."}' | python3 -m json.tool
```

`.env.local` 可选追加：`AI_ENGINE_PRECHECK_TIMEOUT=20`（默认已与代码一致）。
