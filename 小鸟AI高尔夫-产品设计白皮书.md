## 一、执行摘要

### 1.1 一句话定义

**小鸟 AI** 是一款面向中国高尔夫爱好者的 AI 智能教练产品，以微信小程序为主要载体，通过手机摄像头完成挥杆分析、个性化训练指导和 AI 对话答疑，让每位球友"拍一段视频，就能获得专业级教练反馈"。

### 1.2 为什么是现在

| 驱动因素 | 数据支撑 |
|----------|----------|
| 中国高尔夫人口持续增长 | 540 万泛高尔夫人口，核心打球人口 72 万 |
| 室内球馆爆发式增长 | 2,000+ 家球馆，3 年轮次增长 118% |
| 青少年群体快速扩张 | 累计注册 14.1 万人，年增长 12% |
| AI 技术成本骤降 | 视觉模型和中文大模型已达到商用级别，推理成本持续下降 |
| 中国市场产品空白 | 无一款国产产品兼具"深度 AI 分析 + 中文教练 + 训练闭环" |

### 1.3 核心差异化

- **不做「又一个挥杆分析工具」**，做**「中国球友的 AI 私教」**
- 微信小程序优先，**拍视频 → 出报告 → 给训练方案**，三步完成
- 中文高尔夫 LLM 对话教练，用中国教练的语言体系讲解
- B2B（球馆/培训机构）+ B2C（个人用户）双轮驱动

## 二、市场背景与机会分析

### 2.1 全球市场

- AI 高尔夫 App 全球市场规模 2024 年达 **4.12 亿美元**，CAGR **17.2%**
- 预计 2033 年达 **13.7 亿美元**
- 头部产品（Sportsbox AI、GOATCode、Arccos）集中于北美市场
- 亚洲市场仅韩国 GolfFix 跑出规模（100 万下载，1,400 万次分析）

### 2.2 中国市场

#### 高尔夫产业整体数据

| 指标 | 数据 |
|------|------|
| 泛高尔夫人口 | 540 万 |
| 核心打球人口 | 72 万 |
| 球馆参与人口 | 133 万 |
| 营业球场数 | 335 座 |
| 室内球馆数 | 2,000+ 家 |
| 产业总规模 | 151.86 亿元 |
| 户外打球均价 | 531 元/轮 |
| 青少年注册数 | 14.1 万（年增 12%） |

#### 青少年培训市场

- 年度培训费用：**4-5 万元/年/人**
- 单次课时费：**400-1,100 元/小时**
- 家长支付意愿强，对"能量化孩子进步"的工具有明确需求
- 全国主要城市均有高尔夫培训机构覆盖

#### 关键趋势

1. **室内化**：球馆轮次 3 年增长 118%，家庭用户占比从 10% 升至 40%
2. **年轻化**：青少年持续涌入，女性球友比例在球馆场景提升至 30%
3. **数字化需求觉醒**：球友开始习惯用手机拍摄挥杆视频发朋友圈/群，但缺乏分析工具

**技术实现要点：**

| 技术环节 | 方案 |
|----------|------|
| 姿态估计 | MediaPipe Pose / MoveNet + 自训练高尔夫专用模型 |
| 挥杆阶段分割 | 基于关键点时序的分类模型（Setup→Backswing→Top→Downswing→Impact→Follow-through） |
| 动作评分 | 基于角度特征（脊柱角、髋部旋转角、肩转角、手腕角等）的回归模型，参考职业球手数据分布 |
| 问题诊断 | 规则引擎 + ML 分类器识别常见错误模式（over-the-top、early extension、casting 等 12 种） |
| 纠正建议 | RAG 检索高尔夫教学知识库 + LLM 生成个性化建议 |
| 报告生成 | 后端渲染分析报告 → 生成可分享的卡片图 |

#### 功能 2：AI 对话教练

**交互方式：** 类似微信聊天的对话界面

**示例对话：**

```

1. 下杆路径偏外（over-the-top）—— 你最近的挥杆数据显示下杆时
   杆头从外侧切入的角度平均为 4.2°，理想值应在 0-2° 之间。

2. 触球时杆面打开约 3° —— 这和你下杆时过早释放手腕有关。

建议你今天重点做这个练习：
[视频卡片：杆头置于外侧的挥杆路径矫正练习]

做完后再拍一段视频给我看看，我帮你检查改善情况。

用户：上杆的时候身体应该怎么转？

小鸟 AI：上杆时身体旋转的关键要领：

1. 髋部旋转约 45°，肩部旋转约 90°，形成"X-factor"扭力差
2. 感觉像是"背对目标"，右髋保持稳定不侧移
3. 重心移到右脚内侧，但右膝不要外翻

你上次的分析显示你的肩转只有 72°，还有提升空间。
建议对着镜子做慢速转体练习，感受这个幅度差异。
[图片：正确的上杆旋转角度示意图]
```

**技术实现要点：**

| 技术环节 | 方案 |
|----------|------|
| 基础大模型 | 国产中文 LLM（如 DeepSeek / Qwen / GLM）作为基座 |

| RAG 检索增强 | 用户提问 → 检索相关知识 → LLM 结合用户历史数据生成回答 |
| 用户数据绑定 | 对话中自动关联用户历史挥杆数据，实现"懂你"的个性化回答 |
| 多模态回复 | 支持文字 + 图片 + 短视频 + 动画混合回复 |

### 7.2 产品视觉规范（权威，全端统一）

以下规范为 **小鸟 AI** 客户端（微信小程序、React Native App、分享卡片与运营物料）的 **唯一权威视觉标准**。工程实现、外包设计与验收均以此为准；**禁止**在未经过产品评审的情况下，以「更亮/更流行的绿色」单独替代主色体系（例如荧光绿、浅薄荷绿作为主品牌色）。

#### 7.2.1 色彩体系：深绿 + 白 + 金

| 角色 | 用途 | 要求 |
|------|------|------|
| **深绿（主色）** | 主背景/Hero、主导航压色、主按钮常态、关键模块分区 | 色相偏 **冷深绿/墨绿**，联想球场草坪与专业感；避免与高饱和荧光绿混淆 |
| **白（基色）** | 主阅读面、卡片底色、列表行、大面积留白 | 正文与图标需保证 **对比度**（参考 WCAG AA）；白底上的主文字以深灰/近黑为宜 |
| **金（点缀色）** | 会员/付费心智、高分与「出色」等级、关键数据强调、细分割线或图标点缀 | **小面积使用**，避免整屏金色；与深绿并置时注意可读性与无障碍 |

**建议工程色值（设计定稿前可作为开发起点，定稿后以设计 Token 为准）：**

| Token 含义 | 建议 HEX | 备注 |
|--------------|-----------|------|
| 深绿·主色 | `#0f3d2e` | 可作导航栏、主按钮填充 |
| 深绿·渐变终点 | `#1a4d3d` | Hero、头图等与主色渐变搭配 |
| 白·卡片/阅读面 | `#ffffff` | |
| 白·页面浅底（可选） | `#f6f5f0` | 偏暖浅灰白，与金色协调；若需中性灰可用 `#f7f8fa`，但不宜与高饱和绿大面积并置 |
| 金·强调 | `#c9a227`、`#d4af37` | 徽章、关键数字、会员标识 |
| 金·深描边/暗金（可选） | `#8a7020` | 金色描边或深色底上的金色文字 |

#### 7.2.2 应用原则

- **层级**：深绿定调 → 白承载信息与交互 → 金强调「价值节点」（会员、成就、出色评分等）。
- **组件**：主 CTA 可用 **白底深绿字** 或 **深绿底白字**，与页面整体绿白关系二选一保持全站一致；金色不用于主 CTA 填充，以免抢主色语义。
- **数据与状态色**：图表中的成功/警告/错误等 **语义色** 可沿用行业惯例（绿/橙/红），但 **品牌装饰与主界面** 仍以深绿+白+金为框。
- **输出物**：分析报告截图、分享卡片、海报与球馆线下物料，均应在同一套色彩关系下延展，保证线上线下认知一致。

### 7.3 核心页面设计要点

#### 首页

- **视觉风格**：严格遵循 **§7.2 产品视觉规范**（深绿 + 白 + 金）
- **核心 CTA**：屏幕中央大按钮"开始分析"，一目了然
- **信息层级**：最新报告 → 今日任务 → AI 推荐 → 更多功能入口
- **首次进入**：展示一个 15 秒的示例分析视频，让用户秒懂产品

#### 分析报告页

- **沉浸式体验**：全屏播放挥杆视频（骨骼叠加），上滑查看详情
- **可操作性**：每个问题诊断下方直接挂载"开始练习"按钮
- **分享友好**：一键生成高质量社交分享卡片

#### AI 教练对话页

- **亲和力**：给 AI 教练设计一个卡通形象（穿高尔夫服的小鸟）
- **混合回复**：文字 + 图片 + 视频动画 + 可点击的练习卡片
- **快捷入口**：预设常见问题按钮，降低输入门槛
